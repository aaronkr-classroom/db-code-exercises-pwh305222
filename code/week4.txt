Step 2: 요구사항 작성

1. 시스템은 동아리 정보와 회원 정보를 관리해야 한다.

2. 각 동아리는 고유한 동아리명과 개설일을 가진다.

3. 회원은 이름, 학번, 연락처 정보를 포함한다.

4. 특정 동아리에 소속된 회원들의 명단을 조회할 수 있어야 한다.

5. 특정 날짜 이후에 가입한 회원들의 정보를 확인할 수 있어야 한다.

Step 3: 데이터 설계
	
엔티티 1: Clubs (동아리)

- club_id (PK), club_name, created_at

엔티티 2: Members (회원)

- member_id (PK), student_id, name, phone, club_id (FK)

Step 4: SQL 작성

CREATE TABLE Clubs (
    club_id INT PRIMARY KEY,
    club_name VARCHAR(50) NOT NULL,
    created_at DATE
);

CREATE TABLE Members (
    member_id INT PRIMARY KEY,
    student_id VARCHAR(20) UNIQUE,
    name VARCHAR(50),
    phone VARCHAR(20),
    club_id INT,
    joined_at DATE,
    FOREIGN KEY (club_id) REFERENCES Clubs(club_id)
);

SELECT *
FROM Members
WHERE club_id = (
    SELECT club_id 
    FROM Clubs 
    WHERE club_name = '코딩동아리'
)
ORDER BY joined_at DESC;

SELECT name, student_id
FROM Members
WHERE member_id IN (
    SELECT member_id 
    FROM Members 
    WHERE joined_at > '2026-03-04'
)
ORDER BY name ASC;

간단한 설명

동아리(Clubs)와 회원(Members) 두 개의 엔티티를 설계하였습니다. club_id를 외래키로 설정하여 두 테이블 간의 관계를 정의했으며, SELECT 문 실행 시 WHERE 절 내부에 Subquery를 사용하여 특정 동아리에 소속된 인원을 필터링하거나 가입일 조건을 만족하는 데이터를 추출하도록 구현했습니다. 모든 결과는 ORDER BY를 통해 정렬되어 출력됩니다.
